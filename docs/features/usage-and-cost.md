# Usage and cost

Rika derives model cost from Relay `model.usage.reported` events and exposes totals per Turn, Thread, and terminal session. An event-local `cost_usd` or `costUsd` field reported by the provider is authoritative. When no monetary figure is reported, cost is estimated from the event's `input_tokens` and `output_tokens` with a per-model price table, so providers that report only tokens still produce a usable estimate. Generic, nested, or cumulative cost fields are ignored. Child Run usage is attributed to its parent Turn and Thread by traversing Relay's durable child tree.

Each usage cursor is counted once — including on replay, recovery, and out-of-order or duplicate delivery — so cost never double-counts and survives event streams whose sequence numbers regress. Thread totals cover all of that Thread's Turns. The terminal's global total covers Turns found in its bounded load of up to one hundred Threads and falls back to the persisted per-Turn checkpoint sums when live recomputation is unavailable. Transcript pages carry the projected Turn cost and authoritative Thread total; usage reports with neither money nor tokens remain unknown rather than being estimated from transcript text.
